# Blog post template

## Target
600-1800 words, SEO-ready, customer-facing.

## Structure

1. **SEO meta** (top of file, not rendered):
   - `title:` under 60 characters
   - `meta_description:` 140-155 characters
   - `og_title:`, `og_description:`, `og_image:` if different
   - Target keyword (primary) + 2-3 supporting
   - Slug suggestion

2. **Headline (H1)** — sentence case, human, leads with the point. Includes primary keyword naturally.

3. **Lead paragraph** (2-4 sentences) — hook + promise. What the reader gets by reading on.

4. **H2 sections** (3-6 sections typical):
   - Sentence-case headings
   - First paragraph states the point of the section
   - Use concrete examples, specific numbers, direct quotes where available
   - Short paragraphs, vary length
   - Internal links to product / collection / other blog pages (respect project link conventions, e.g. `/en/` prefix for EN)

5. **Closing** — summary + clear next action (link to product, newsletter signup, related reading). Not "In conclusion".

6. **Image suggestions** — list of 2-4 image spots with alt-text drafts. Reference available assets when possible.

## Bilingual notes (if applicable)

- EN version is NOT a literal translation — it's a parallel version tuned for the EN audience
- EN uses international varietal names (e.g. "Pinot Blanc" not "Rulandské biele")
- SK keeps Slovak terminology (e.g. "víno s prívlastkom neskorý zber")
- URL slugs typically match but may differ if a localized phrase works better

## Quality checklist (self-verify before writing)

- Does the lead promise something concrete?
- Is every factual claim verifiable from public sources?
- Are internal links correct for the project's locale convention?
- Does the closing have a clear next action that's NOT generic?
- Any AI-isms? Em-dashes? Generic intensifiers?
- Word count within range?
